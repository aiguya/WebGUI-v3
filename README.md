# WebGUI.v3 사용 매뉴얼

WebGUI.v3은 로컬 PC에서 실행하는 AI 이미지/영상 생성 WebUI입니다. Grok/xAI 계열 생성, Hermes xAI OAuth 프록시, Codex/ChatGPT OAuth 이미지 프록시, 로컬 라이브러리 관리, 영상 연장, 영상 편집, 망가 실사화/역식 배치 작업을 한 화면에서 다루도록 만든 Windows 중심 앱입니다.

기본 실행 주소는 다음과 같습니다.

```text
http://127.0.0.1:7863
```

## 빠른 실행

### 독립 앱 창으로 실행

가장 편한 방식입니다.

```text
run_webgork_app.bat
```

동작:

- 필요한 Python 패키지가 없으면 자동 설치합니다.
- 서버가 꺼져 있으면 `127.0.0.1:7863`로 실행합니다.
- Chrome이 있으면 `--app=` 모드의 독립 창으로 엽니다.
- Chrome을 찾지 못하면 기본 브라우저로 엽니다.

### 일반 브라우저로 실행

```text
run_webgork.bat
```

콘솔 창이 열린 상태로 서버가 실행됩니다. 종료하려면 콘솔에서 `Ctrl+C`를 누르거나 창을 닫습니다.

### 수동 실행

Python 3.10 이상이 필요합니다.

```powershell
python -m pip install -r requirements.txt
set WEBGORK_PORT=7863
python app.py
```

## 화면 구성

상단은 두 줄 헤더입니다.

- 첫 줄: `WebGUI.v3` 타이틀, Hermes 연결 상태, Codex 이미지 프록시 연결 상태, credit 잔량 표시
- 둘째 줄: 아이콘 탭 메뉴

메인 화면은 세 영역입니다.

- 왼쪽: 세로형 작업 큐
- 가운데: 결과 미리보기
- 오른쪽: 현재 탭의 컨트롤 패널

결과 미리보기는 이미지/영상이 패널 밖으로 넘치지 않게 맞춰집니다. 우측 컨트롤 패널만 세로 스크롤되고, 큐는 좌측에서 독립적으로 스크롤됩니다.

## 프로젝트 작업공간

특정 테마나 작업 묶음을 프로젝트로 만들고, 이후 생성되는 결과물에 자동으로 프로젝트 메타데이터를 남길 수 있습니다.

지원:

- 상단 헤더에서 현재 프로젝트 선택
- 설정 탭에서 프로젝트 생성, 수정, 삭제, 즐겨찾기
- 프로젝트 제목, 설명, 태그 저장
- 이미지 생성/편집, 이미지->영상, 공식 연장, 프레임 연장, 영상 편집, 망가 배치 결과물에 프로젝트 정보 기록
- 라이브러리에서 프로젝트별 결과물 필터링

프로젝트를 선택하지 않으면 기존처럼 프로젝트 없는 결과물로 저장됩니다. 프로젝트를 삭제해도 이미 생성된 파일은 삭제되지 않으며, 결과물에 남은 프로젝트 제목으로 라이브러리에서 계속 식별할 수 있습니다.

## Provider와 인증

### Hermes xAI OAuth 프록시

Grok/xAI OAuth 기반 사용을 위한 프록시입니다. 앱은 Hermes가 제공하는 로컬 proxy endpoint를 호출합니다.

기본 프록시 주소:

```text
http://127.0.0.1:8645
```

설정 탭에서 할 수 있는 일:

- Hermes 인증 시작
- 인증 코드 붙여넣기
- Hermes 로그아웃
- Hermes 프록시 시작
- 연결 상태 확인

주의:

- OAuth 승인 자체는 사용자가 브라우저에서 직접 해야 합니다.
- 앱은 공유 client_id나 개인 토큰을 배포본에 포함하지 않습니다.
- Hermes 프록시가 `/v1/images/generations`, `/v1/images/edits`, `/v1/videos/generations`, `/v1/videos/extensions` 같은 경로를 지원하지 않으면 해당 기능은 실패할 수 있습니다.

### Codex/ChatGPT OAuth 이미지 프록시

GPT 계열 이미지 생성/편집을 로컬 프록시를 통해 사용하는 선택 기능입니다.

설정 탭에서 할 수 있는 일:

- Codex 이미지 프록시 상태 확인
- Codex 이미지 프록시 시작

주의:

- 프록시 실행 파일이나 `npx`/Node 환경이 없으면 시작할 수 없습니다.
- GPT 모델을 선택했는데 프록시가 실패하면 Grok으로 자동 전환하지 않고 오류를 표시합니다.
- ChatGPT Pro 구독과 OpenAI API 과금/프록시 사용 조건은 별개일 수 있습니다. 사용하는 프록시의 정책을 확인해야 합니다.

### Credit 표시

헤더 오른쪽에 credit 잔량이 배터리 바 형태로 표시됩니다.

- 기본: 초록색
- 10% 이하: 주황색
- 5% 이하: 붉은색

가능하면 OAuth 세션으로 다음 billing endpoint를 조회합니다.

```text
https://cli-chat-proxy.grok.com/v1/billing
```

조회에 성공하면 `config.monthlyLimit`, `config.used` 값으로 잔량 퍼센트를 계산합니다. 실패하면 설정 화면의 usage 연결 버튼으로 Grok 공식 usage 화면에서 확인합니다.

credit 상태는 다음 시점에 갱신됩니다.

- 앱 새로고침
- Provider 상태 갱신
- 생성/편집 결과 수신 후
- 설정 화면에서 수동 확인

## 주요 기능

### 이미지 생성

텍스트 프롬프트로 이미지를 생성합니다.

지원 UI:

- provider/model 선택
- 결과 비율 선택: `auto`, `원본`, `2:3`, `3:2`, `1:1`, `9:16`, `16:9`
- Grok 해상도 옵션: 가능 조합에서 `1k`, `2k`
- 오류 발생 시 상세 로그 팝업 표시

선택한 provider/model 조합이 실패하면 임의로 다른 provider로 바꾸지 않습니다.

### 이미지 편집

이미지를 참조해서 편집합니다.

입력 방식:

- 파일 업로드
- 드래그앤드랍
- 클립보드 붙여넣기
- 라이브러리에서 불러오기

다중 이미지:

- 첫 번째 이미지는 메인 이미지로 표시됩니다.
- 추가 이미지는 오른쪽 세로 썸네일로 표시됩니다.
- 썸네일의 `X` 버튼으로 제거할 수 있습니다.
- 썸네일 클릭 시 확대해서 볼 수 있습니다.

처리 방식:

- 여러 이미지를 한 장으로 이어붙인 뒤 편집
- 여러 이미지를 API에 다중 첨부해서 편집

추가 옵션:

- 결과 비율 선택
- 실험 해상도: `auto`, `1k`, `2k`
- 초기화 버튼: 입력 이미지와 결과 미리보기를 함께 초기화

### 이미지 -> 영상

이미지를 참조해서 영상을 생성합니다.

지원 UI:

- 이미지 업로드/붙여넣기/라이브러리 불러오기
- 프롬프트 입력
- 모델 선택
- 해상도 선택
- 결과 비율 선택
- `2k 업스케일 후 영상 생성` 체크박스

업스케일 체크박스를 켜면 먼저 이미지 편집의 2k 업스케일 흐름을 사용한 뒤 영상 생성을 요청합니다. 체크하지 않으면 선택한 원본 이미지만 사용합니다.

`grok-imagine-video-1.5-preview` 모델은 레퍼런스 이미지를 1장만 사용합니다. 이 모델을 선택하면 기존에 여러 장이 올라가 있어도 첫 번째 이미지만 남고, 이 상태에서 새 이미지를 추가하면 기존 레퍼런스가 새 이미지로 교체됩니다.

### 공식 영상 연장

xAI/Grok 공식 영상 연장 흐름을 사용합니다.

특징:

- 영상 파일 업로드 또는 라이브러리에서 선택
- 연장 길이 선택
- 모델 선택
- 공식 연장에서는 해상도 선택을 표시하지 않습니다.

15초 이하 입력 영상:

- 서버가 반환한 결과물을 그대로 저장합니다.
- 별도의 로컬 이어붙이기 후처리를 하지 않습니다.

15초 초과 입력 영상:

1. 앞부분과 마지막 15초를 분리합니다.
2. 마지막 15초만 공식 연장 요청에 사용합니다.
3. 서버에서 받은 연장 결과 앞에 잘라두었던 앞부분을 다시 붙입니다.
4. 최종 결과를 라이브러리에 저장합니다.

공식 연장 결과가 무음이거나 비율이 바뀌는 경우는 서버 반환값 또는 로컬 후처리 조건을 확인해야 합니다.

### 프레임 연장

로컬에서 마지막 프레임을 기준으로 새 영상을 만들고 원본 뒤에 이어붙이는 기능입니다.

현재 동작:

1. 원본 영상의 마지막 프레임을 캡처합니다.
2. `마지막 프레임 2K 업스케일` 옵션이 켜져 있으면 마지막 프레임을 2k 업스케일합니다.
3. 준비된 마지막 프레임으로 새 영상을 생성합니다.
4. 원본 영상 뒤에 새 영상을 붙여 최종 영상을 저장합니다.

옵션:

- 모델 선택
- 해상도 선택
- 결과 비율 선택
- 마지막 프레임 2K 업스케일 체크박스
- 음소거 체크박스

마지막 프레임 업스케일에 사용하는 프롬프트 템플릿:

```text
Upscale this extracted last frame to 2K quality for use as the first frame of a continuation video. Preserve the exact composition, identity, pose, clothing, lighting, color, camera angle, and background. Do not add, remove, crop, reinterpret, or apply motion. Continuation prompt for context only: {사용자가 입력한 연장 프롬프트}
```

음소거를 끄면 가능한 범위에서 원본/연장 구간 오디오를 보존합니다. 단, 생성 provider가 무음 영상을 반환하면 새 구간 오디오는 없을 수 있습니다.

### 그림 프롬프트

이미지를 분석해서 프롬프트를 추출합니다.

지원:

- 파일 업로드
- 라이브러리 이미지 선택
- 추출 결과 복사

결과 패널은 텍스트 확인과 복사에 필요한 크기로 표시됩니다.

### 프롬프트 관리

작업용 프롬프트를 별도 자산으로 저장하고 다시 사용할 수 있습니다.

지원:

- 프롬프트 저장/수정/삭제
- 즐겨찾기
- 작업 종류별 분류: 이미지 생성, 이미지 편집, 이미지→영상, 공식 연장, 프레임 연장, 망가 실사화·역식, 범용
- 태그와 검색
- 구조형 입력: 대상, 장면, 스타일, 조명, 카메라, 유지, 변경, 제외, 추가 지시
- 구조형 입력을 최종 프롬프트로 자동 합성
- 저장된 프롬프트를 현재 작업 탭으로 보내기
- 라이브러리 결과물의 프롬프트를 프롬프트 관리 탭에 저장
- 같은 프롬프트를 수정하면 이전 내용을 버전 기록으로 보존

저장 파일:

```text
저장루트/prompts.json
```

### 망가 실사화 + 역식

망가/만화 이미지를 대량으로 처리하는 배치 기능입니다.

지원:

- 최대 500장 등록
- 최대 50개 병렬 변환
- 항목별 등록 취소
- 전체 취소
- 실제 완료 개수 기반 진행률 표시
- 컷 분리 실사화 모드
- 캐릭터/스타일 레퍼런스 최대 2장 첨부

파일명:

```text
grok_trans_원본파일명.png
```

순서를 잃지 않도록 원본 파일명을 보존합니다. 숫자 prefix가 앞에 붙어 순서가 뒤섞이지 않게 구성되어 있습니다.

컷 분리 실사화 모드는 업로드한 만화 페이지를 흰 여백/컷 경계 기준으로 자동 분리한 뒤, 각 컷을 별도 이미지로 실사화합니다. 캐릭터/스타일 레퍼런스를 첨부하면 컷 이미지와 함께 참조 이미지로 전달됩니다.

컷 분리 실사화 결과 파일명:

```text
grok_panel_원본파일명_p001_c001_생성ID.png
```

중간 컷 crop은 `manga_panels` 폴더에 저장되고, 최종 실사화 결과는 `manga_trans` 폴더에 저장됩니다.

저장 위치:

```text
저장루트/manga_trans
```

결과는 라이브러리에도 표시됩니다.

### 영상 편집

업로드 영상이나 라이브러리 영상을 간단히 자르고 붙이는 로컬 편집 기능입니다.

지원:

- 여러 영상 추가
- 라이브러리에서 불러오기
- 클립 순서 변경
- 드래그로 순서 재배치
- 위/아래 이동 버튼
- 클립별 시작/끝 시간 입력
- 현재 재생 위치를 시작/끝으로 지정
- fade in/out
- crossfade 전환
- 음소거
- 오디오 보존

처리는 FFmpeg를 사용합니다. `imageio-ffmpeg`가 포함되어 있어 별도 FFmpeg가 없어도 대부분 동작하지만, 시스템 FFmpeg가 있으면 우선 사용합니다.

## 템플릿 포맷 호환 정책

현재 영상 템플릿과 재사용 블록 저장 구조는 1차 포맷으로 고정되어 있습니다.

저장되는 JSON에는 다음 값이 포함됩니다.

```json
{
  "format_version": 1
}
```

호환 원칙:

- `format_version`이 없는 기존 템플릿/블록은 v1로 간주합니다.
- 앱은 템플릿/블록을 읽을 때 먼저 포맷 마이그레이션 함수를 통과시킵니다.
- 저장할 때는 현재 앱이 지원하는 최신 포맷으로 다시 저장합니다.
- 이후 v2, v3 구조가 추가되더라도 `v1 -> v2 -> v3` 순서의 변환 함수를 추가해 이전 파일을 계속 읽을 수 있게 유지합니다.

## 작업 큐

일반 작업 큐는 최대 30개 병렬 처리를 기준으로 동작합니다.

큐에는 다음 정보가 표시됩니다.

- 썸네일
- 작업 종류
- 상태
- 진행률
- 완료/실패 작업 정리 버튼

서버에서 실제 진행률을 알 수 있는 작업은 실제 값을 표시합니다. 진행률을 알 수 없는 작업은 진행 중임을 알 수 있는 UI로 표시합니다.

## 라이브러리

생성 결과와 업로드 원본을 관리합니다.

지원:

- 전체/이미지/영상 필터
- 프로젝트 필터
- 프롬프트/파일명/모델 검색
- 작업 종류 필터
- 날짜 필터: 오늘, 어제, 최근 7일, 이번 달, 날짜 선택
- 정렬: 최신순, 오래된순, 파일명순, 즐겨찾기 우선, 해상도순
- 보기 방식: 그리드, 타임라인, 작업 체인
- 썸네일 크기: 작게, 보통, 크게

- 즐겨찾기

- 선택 삭제
- 개별 삭제

- 파일 복사

- 프롬프트 복사
- 이미지 확대 보기
- 영상 재생

프롬프트 표시:

- 카드에서는 최대 3줄까지만 표시합니다.
- 나머지는 말줄임 처리합니다.
- 프롬프트 영역을 클릭하면 테두리 없는 모달로 전체 프롬프트를 볼 수 있습니다.
- 모달 안 텍스트는 블록 선택이 가능하고, 아래에 `프롬프트 복사` 버튼이 있습니다.

영상 썸네일:

- 영상의 약 1초 지점 프레임을 썸네일로 사용합니다.
- 라이브러리 선택 모달에서는 1:1 비율 썸네일로 표시합니다.
- 재생 모달에서는 영상이 자동 반복 재생됩니다.

즐겨찾기:

- 별 아이콘으로 표시됩니다.
- 선택됨/선택 안 됨 상태가 다르게 보입니다.
- 상태는 metadata와 함께 영구 저장됩니다.

결과물이 많을 때 예전 영상을 빠르게 찾으려면 `영상` 필터와 `오래된순` 정렬을 함께 쓰거나, `타임라인` 보기로 날짜별 목록을 확인하는 방식이 가장 빠릅니다. 원본 이미지에서 영상, 연장, 편집 결과까지 이어진 흐름은 `작업 체인` 보기에서 묶어서 확인할 수 있습니다.

## 저장 경로

설정 탭에서 결과물 저장 폴더를 지정할 수 있습니다.

지원:

- 직접 경로 입력
- Windows 파일 브라우저로 선택
- 저장 경로 적용
- 기존 라이브러리 이동

저장 경로 아래에는 다음 폴더가 자동 생성됩니다.

```text
image
video
uploads
thumbnails
manga_trans
manga_panels
metadata-backups
```

저장 경로를 변경하면 기존 라이브러리 파일과 메타데이터를 가능한 범위에서 함께 이동합니다.

## 오류 로그

오류가 발생하면 팝업에 다음 정보가 표시됩니다.

- 발생 시간
- 사용자용 메시지
- 서버/provider 응답 detail

복사해서 디버깅에 사용할 수 있습니다.

자주 보이는 오류:

- `CONTENT_POLICY_VIOLATION`: provider의 콘텐츠 정책 검열에 걸린 경우
- `Video extension is not supported for this model`: 선택 모델이 영상 연장을 지원하지 않는 경우
- `reference_images is not supported for this model`: 선택 모델이 참조 이미지를 지원하지 않는 경우
- `internal_error`: provider 서버 내부 실패
- `path_not_allowed`: 프록시가 해당 API 경로를 전달하지 않는 경우
- `Content Too Large`: 업로드 데이터가 프록시 또는 서버 제한을 넘은 경우
- `Failed to fetch`: 프론트엔드에서 서버 연결이 끊겼거나 요청이 차단된 경우

## 배포

다른 PC에 전달할 때는 `release` 폴더만 압축해서 전달하는 것을 권장합니다.

포함되는 항목:

- `app.py`
- `requirements.txt`
- `README.md`
- `CODEX_REBUILD_RECIPE.md`
- `.env.example`
- `run_webgork.bat`
- `run_webgork_app.bat`
- `static/`
- `templates/`

포함하면 안 되는 항목:

- `.webgork-private/`
- OAuth 토큰
- API 키
- 세션 쿠키
- 개인 media 결과물
- 개인 저장 경로 설정
- 로그 파일
- `media-library/`
- `__pycache__/`

배포받는 사람은 자신의 PC에서 직접 인증해야 합니다.

## 원클릭 설치 범위

배치 파일로 자동화 가능한 것:

- Python 패키지 설치
- WebGUI 서버 실행
- Chrome app 창 열기
- Hermes/Codex 프록시 실행 시도
- 프록시 상태 확인

자동화할 수 없는 것:

- 사용자의 OAuth 승인
- xAI/Grok 계정 로그인
- ChatGPT/Codex 계정 로그인
- 외부 서비스 약관 동의

보안상 OAuth 승인 화면은 사용자가 직접 확인하고 진행해야 합니다.

## 개발/검증

문법 확인:

```powershell
python -m py_compile app.py
```

헬스체크:

```powershell
python -c "from app import app; c=app.test_client(); print(c.get('/health').status_code, c.get('/health').json)"
```

브라우저 캐시가 의심될 때:

- 새로고침
- 강력 새로고침
- 서비스워커 업데이트 확인
- `templates/index.html`과 `static/service-worker.js`의 정적 파일 버전 확인
