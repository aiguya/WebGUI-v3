# WebGrok v3 Hermes-only 사용자 매뉴얼

이 문서는 `WebGrok-v3-Hermes` 릴리즈 전달본을 받은 사용자를 위한 기능 설명서입니다.

이미지 생성, 이미지 편집, 이미지→영상, 영상 관련 요청은 기본적으로 로컬 Hermes Agent Proxy endpoint를 통해 처리됩니다. 첫 실행 시 필요한 의존성은 릴리즈 실행 파일이 자동 준비를 시도합니다.

## 1. 릴리즈 구성

릴리즈 폴더의 주요 파일은 다음과 같습니다.

- `WEBGROK_CHROME_APP.exe`: Chrome 앱 모드로 WebGrok을 여는 원클릭 실행 파일입니다.
- `RUN_WEBGROK_HERMES_ONLY.bat`: 기본 브라우저로 WebGrok을 여는 실행 파일입니다.
- `WEBGROK_BOOTSTRAP.bat`: 첫 실행에 필요한 앱 Python 패키지와 Hermes Agent를 준비하는 설치 스크립트입니다. 사용 가능한 Python 3.11 이상이나 Hermes Agent가 이미 있으면 기존 설치를 재사용합니다.
- `README_RELEASE.md`: 릴리즈 구성과 첫 실행 요약입니다.
- `USER_MANUAL.md`: 현재 문서입니다.
- `webgork-settings.json`: 기본 provider 설정입니다.
- `media-library/`: 생성 결과, 프로젝트, 프롬프트, 템플릿 데이터가 저장되는 폴더입니다.

릴리즈에는 개발자의 생성 이미지, 영상, 개인 설정, 인증 토큰, 로컬 로그가 포함되지 않습니다.

## 2. 첫 실행

1. zip 파일을 원하는 폴더에 압축 해제합니다.
2. `WEBGROK_CHROME_APP.exe`를 실행합니다.
3. 처음 실행하는 PC라면 설치 창이 열리고 앱 Python 패키지를 준비합니다. 기존 Hermes Agent가 있으면 재사용하고, 없을 때만 릴리즈 폴더의 `.hermes-venv`에 Hermes Agent를 설치합니다. 사용 가능한 Python 3.11 이상이 없을 때만 `winget`으로 Python 설치를 시도합니다.
4. Node.js가 없으면 Codex/ChatGPT OAuth Proxy용으로 `winget` 설치를 시도합니다. 이 단계가 실패해도 Hermes 기본 기능은 실행할 수 있습니다.
5. 앱이 열리면 설정 화면의 `Hermes xAI` 행에서 `인증`을 눌러 Hermes xAI OAuth를 연결합니다.
6. 브라우저가 열리지 않으면 `RUN_WEBGROK_HERMES_ONLY.bat`를 실행합니다.

기본 주소는 `http://127.0.0.1:7863`입니다.

### 필요한 구성요소

릴리즈 앱은 다음 구성요소를 필요로 하며, 첫 실행 시 자동 준비를 시도합니다.

- Python 3.11 이상: WebGrok 서버와 Hermes Agent 실행에 필요합니다. 이미 설치되어 있으면 새로 설치하지 않고 재사용합니다.
- 인터넷 연결: Python 패키지, Hermes Agent, 선택적으로 Node.js를 내려받는 데 필요합니다.
- Hermes Agent: 기존 Hermes Agent가 있으면 재사용합니다. 없을 때만 릴리즈 폴더의 `.hermes-venv`에 자동 설치됩니다.
- Node.js/npx: Codex/ChatGPT OAuth Proxy 기능에 필요합니다. Hermes 기능만 사용할 때는 Node.js 설치 실패가 앱 실행을 막지 않습니다.
- Chrome: `WEBGROK_CHROME_APP.exe`의 앱 모드 실행에 필요합니다. Chrome이 없으면 기본 브라우저로 열립니다.

### 설치/실행 실패 확인

설치 또는 실행이 실패하면 실패 화면에 최근 로그가 함께 표시됩니다.

- 설치 실패 상세: `work/bootstrap.log`
- 서버 실행 실패 상세: `work/server-runner.log`

대표적인 실패 원인은 `winget` 없음, 인터넷 차단, pip 패키지 설치 실패, Hermes Agent 탐지/설치 실패, Python 탐지/PATH 문제입니다. WindowsApps의 `python.exe`/`py.exe` 별칭은 실제 Python이 아니면 무시됩니다. 재사용된 Hermes Agent 경로는 `work/hermes-exe.txt`에 기록됩니다.

이미 `127.0.0.1:7863`에서 이전 WebGrok 서버가 실행 중이면 새 릴리즈 런처가 경고를 표시할 수 있습니다. 이 경우 기존 WebGrok 서버를 종료한 뒤 다시 실행하세요.

## 3. 설정

설정 탭에서는 연결과 모델 관련 설정을 관리합니다.

### 연결 상태

- `Hermes xAI`: Hermes OAuth와 Proxy 상태를 보여줍니다.
- `Codex / ChatGPT`: Codex OAuth Proxy 상태를 보여줍니다.
- `Codex / ChatGPT`의 `로그아웃`은 로컬 Codex OAuth 세션을 삭제하고 앱이 띄운 Codex Proxy를 종료합니다.

Hermes를 사용할 때는 `Hermes xAI`가 연결되어 있고, Proxy가 실행 중이어야 합니다.

### Provider 설정

- `호출 방식`: 기본값은 `Hermes Agent Proxy`입니다.
- `Hermes Proxy Base URL`: Hermes endpoint 주소입니다. 기본값은 `http://127.0.0.1:8645/v1`입니다.
- `Hermes Proxy Token`: 필요한 경우에만 입력합니다.
- `Provider 저장`: 설정을 저장합니다.
- `Hermes 연결 테스트`: 현재 endpoint로 연결 가능한지 확인합니다.

### Hermes 모델 검색

이미지 생성, 이미지 편집, 영상 등 범위별로 후보 모델을 실제 요청으로 확인합니다.

주의: 모델 검색은 실제 Hermes/Grok 요청량을 사용할 수 있습니다. 많이 실행하지 말고 필요한 후보만 확인하는 용도로 사용하세요.

## 4. 공통 사용법

대부분의 작업 탭은 비슷한 흐름으로 동작합니다.

1. 요청 경로 또는 Provider를 선택합니다.
2. 모델과 해상도, 길이, 수량 같은 옵션을 설정합니다.
3. 프롬프트를 입력합니다.
4. 필요한 경우 파일을 업로드하거나 라이브러리에서 불러옵니다.
5. 실행 버튼을 누릅니다.
6. 작업 큐와 미리보기에서 진행 상태와 결과를 확인합니다.

결과물은 자동으로 `media-library/`에 저장되고 라이브러리에 등록됩니다.

## 5. 이미지 생성

텍스트 프롬프트로 이미지를 생성하는 기능입니다.

주요 입력:

- Provider: Hermes Proxy 또는 Codex/ChatGPT 경로를 선택합니다.
- 모델: 선택한 Provider에서 사용 가능한 이미지 모델을 선택합니다.
- 해상도: 지원되는 해상도를 선택합니다.
- 프롬프트: 생성하고 싶은 이미지 내용을 입력합니다.
- 동시 생성 수량: 같은 프롬프트로 여러 장을 큐에 넣을 수 있습니다.

생성된 결과는 미리보기와 라이브러리에 표시됩니다. 미리보기 전체화면에서 `편집`, `영상` 버튼을 누르면 해당 결과를 바로 이미지 편집 또는 이미지→영상 탭으로 보낼 수 있습니다.

## 6. 이미지 편집

참조 이미지를 기반으로 새 이미지를 만드는 기능입니다.

참조 이미지 입력 방식:

- 로컬 파일 업로드
- 라이브러리에서 불러오기
- 미리보기 전체화면의 `편집` 버튼으로 바로 전달

사용 흐름:

1. 편집할 이미지를 선택합니다.
2. 편집 프롬프트를 입력합니다.
3. 모델과 옵션을 선택합니다.
4. 실행합니다.

기존 프롬프트가 입력되어 있으면, 미리보기에서 이미지를 보내도 프롬프트는 유지되고 참조 이미지만 교체됩니다.

## 7. 이미지→영상

이미지를 시작점으로 영상을 생성하는 기능입니다.

주요 옵션:

- 참조 이미지: 파일 업로드 또는 라이브러리에서 불러오기
- 프롬프트: 영상의 움직임, 카메라, 분위기, 장면 변화를 입력합니다.
- 영상 모델: Hermes Proxy에서 사용 가능한 영상 모델을 선택합니다.
- 길이/해상도: 지원되는 범위 안에서 선택합니다.
- 동시 생성 수량: 같은 이미지와 프롬프트로 여러 요청을 큐에 넣습니다.

여러 이미지를 각각 별도 요청으로 큐에 넣는 옵션을 켜면, 라이브러리에서 최대 10개의 이미지를 선택할 수 있습니다. 예를 들어 이미지 5개와 동시 생성 수량 10을 사용하면 총 50개의 영상 요청이 큐에 등록됩니다.

## 8. 영상 연장과 프레임 연장

영상 연장 기능은 Hermes Proxy 경로를 기준으로 동작합니다.

- 영상 연장: 기존 영상을 이어서 생성합니다.
- 프레임 연장: 선택한 프레임 또는 구간을 기준으로 이어지는 영상을 생성합니다.

모델 지원 여부는 Hermes Proxy와 연결된 백엔드 상태에 따라 달라집니다. 요청이 실패하면 설정의 Hermes 모델 검색에서 사용 가능한 영상 모델을 확인하세요.

## 9. 영상 편집

기존 영상과 프롬프트를 기반으로 수정 영상을 요청하는 기능입니다.

사용 흐름:

1. 영상 파일을 업로드하거나 라이브러리에서 불러옵니다.
2. 수정 방향을 프롬프트로 입력합니다.
3. 모델과 옵션을 선택합니다.
4. 실행 후 큐와 미리보기에서 결과를 확인합니다.

## 10. 망가 실사화·역식

여러 이미지 파일을 batch로 처리하는 기능입니다.

주요 용도:

- 이미지 번역/역식 흐름
- 패널 단위 실사화 또는 재구성 흐름
- 여러 장을 한 번에 큐에 등록하는 배치 작업

파일 수가 많으면 처리 시간이 길어질 수 있습니다. 먼저 적은 수량으로 테스트한 뒤 batch 크기를 늘리는 방식이 좋습니다.

## 11. 그림 프롬프트

이미지를 분석해 프롬프트를 추출하는 기능입니다.

사용 흐름:

1. 이미지를 업로드하거나 라이브러리에서 선택합니다.
2. 프롬프트 추출을 실행합니다.
3. 결과 프롬프트를 복사하거나 프롬프트 탭에 저장합니다.

프롬프트 추출도 Provider 연결 상태에 따라 동작합니다. Hermes 또는 Codex Proxy 연결이 필요할 수 있습니다.

## 12. 프롬프트 탭

자주 쓰는 프롬프트를 저장하고 다시 불러오는 공간입니다.

주요 기능:

- 프롬프트 저장
- 즐겨찾기
- 검색
- 작업 유형별 분류
- 생성/편집/영상 탭에서 재사용

반복 작업을 많이 한다면 프롬프트를 먼저 저장해두고 각 기능 탭에서 불러와 사용하는 방식이 편합니다.

## 13. 영상 템플릿 탭

여러 단계의 영상 제작 흐름을 템플릿으로 구성하는 기능입니다.

주요 개념:

- 변수: 프롬프트 안에서 반복해서 바꿀 값입니다.
- 슬롯: 이미지나 영상 같은 입력 자료를 넣는 자리입니다.
- 샷: 실제 실행되는 단계입니다.
- 블록: 이미지 생성, 편집, 영상화 같은 기능 단위 설정을 저장한 조각입니다.

기본 흐름:

1. 템플릿을 선택하거나 새로 만듭니다.
2. 변수와 슬롯을 설정합니다.
3. 샷을 추가하고 각 샷의 작업 방식을 정합니다.
4. 실행 전 미리보기를 확인합니다.
5. `템플릿 큐에 등록`으로 작업을 실행합니다.

릴리즈에는 정리된 샘플 템플릿 JSON이 포함됩니다. 이미지나 영상 샘플 파일은 포함되지 않습니다.

## 14. 라이브러리

생성 결과와 업로드 기반 결과를 관리하는 화면입니다.

주요 기능:

- 이미지/영상/즐겨찾기/템플릿 결과 필터
- 검색
- 날짜 필터
- 정렬
- 썸네일 크기 변경
- 즐겨찾기
- 파일 열기
- 파일 복사
- 삭제
- 전체화면 미리보기

이미지 전체화면 미리보기에서는 `편집`, `영상` 버튼으로 라이브러리를 거치지 않고 바로 다음 작업 탭으로 보낼 수 있습니다.

최근 버전에서는 라이브러리 불러오기 선택창이 경량 API와 캐시를 사용합니다. 항목이 많아도 이전보다 훨씬 빠르게 열립니다.

## 15. 작업 큐

실행한 작업은 우측 작업 큐에 표시됩니다.

큐에서 확인할 수 있는 정보:

- 실행 중/대기/완료/실패 상태
- 작업 이름과 프롬프트 요약
- 결과 보기
- 오류 확인

동시 생성 수량은 최대 100개까지 입력할 수 있습니다. 많은 수량을 넣으면 Hermes/Grok 요청량을 빠르게 사용할 수 있으니 먼저 작은 수량으로 테스트하는 것을 권장합니다.

## 16. 오류 확인

실패한 작업은 오류 메시지와 상세 로그를 제공합니다.

자주 확인할 항목:

- Hermes Proxy가 실행 중인지
- `Hermes Proxy Base URL`이 맞는지
- Hermes OAuth가 연결되어 있는지
- 선택한 모델이 현재 Hermes에서 사용 가능한지
- 입력 파일이 실제로 존재하는지
- 영상/이미지 URL이 404가 아닌지

검열 또는 moderation 관련 실패는 작업 결과가 생성되지 않을 수 있습니다. 릴리즈 앱은 이런 경우를 사용자가 확인할 수 있도록 오류 또는 토스트로 안내합니다.

## 17. 데이터 위치와 백업

기본 데이터는 릴리즈 폴더의 `media-library/` 아래에 저장됩니다.

중요 파일:

- `metadata.json`: 라이브러리 목록
- `prompts.json`: 프롬프트 저장소
- `projects.json`: 프로젝트 목록
- `video-templates.json`: 영상 템플릿
- `video-template-blocks.json`: 템플릿 블록

릴리즈 폴더를 통째로 백업하면 결과물과 설정을 함께 보관할 수 있습니다. 다른 사람에게 공유할 때는 `media-library/` 안의 개인 생성물과 설정 파일에 민감한 내용이 없는지 확인하세요.
